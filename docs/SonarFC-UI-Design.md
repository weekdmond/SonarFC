# SonarFC UI 设计稿说明

**版本**：v1.0  
**日期**：2026-04-06  
**状态**：Draft  
**目标参考**：保留 SonarFC 原版原型的布局和字体层级，只收掉紫色科技感，改成更干净的深色产品配色

---

## 1. 设计目标

SonarFC 的页面不应该像交易终端，也不应该像炫技仪表盘。它要像一个成熟的体育信息产品：

- 第一眼就看懂
- 重点信息在首屏
- 色彩只负责提示，不喧宾夺主
- 页面层级稳定，用户能快速扫读多场比赛

核心关键词：

- 简洁
- 可信
- 体育感
- 信息优先
- 移动端友好

---

## 2. 视觉方向

### 2.1 总体风格

- 保留原版深色主界面和高信息密度布局，不重做结构
- 用绿色替代原先偏紫的品牌强调色，让页面更稳、更干净
- 面板保持深色分层，主要靠边框、亮度差和少量团队色区分信息
- 视觉重点仍然放在排版、对比和状态标签，而不是装饰性渐变

### 2.2 设计原则

- 一个区块只表达一件事
- 颜色只用于状态，不用于装饰
- 重要数字配短标签，不单独裸露
- 比赛详情页优先展示对比，而不是堆叠描述
- 所有复杂模块都先做“可扫读版”，再做“深挖版”

## 3. 配色系统

### 3.1 品牌色

| 角色 | 色值 | 用途 |
|------|------|------|
| Brand Primary | `#18C37D` | 顶部品牌、选中态、主按钮 |
| Brand Alt | `#6EE7B7` | 品牌渐变的浅端、轻强调 |
| Brand Soft | `#18C37D1A` | 选中 tab、弱高亮背景 |

### 3.2 中性色

| 角色 | 色值 | 用途 |
|------|------|------|
| App Background Top | `#060A08` | 页面顶部背景 |
| App Background Mid | `#0B120F` | 页面主体背景 |
| Surface | `#0F1512` | 主卡片背景 |
| Surface Alt | `#171D19` | 卡片渐变深端、弹窗背景 |
| Border | `rgba(255,255,255,0.08)` | 卡片边框、分割线 |
| Text Primary | `#FFFFFF` | 主标题、主信息 |
| Text Secondary | `rgba(255,255,255,0.46)` | 辅助说明、时间、标签 |
| Text Faint | `rgba(255,255,255,0.20)` | 次弱提示 |

### 3.3 状态色

| 状态 | 色值 | 用途 |
|------|------|------|
| Positive | `#178D5B` | 体能充沛、阵容完整、轻压力 |
| Warning | `#C98512` | 中等疲劳、赛程偏密 |
| Danger | `#D14F45` | 高疲劳、缺阵较多、旅行负担高 |
| Info | `#2F6FDE` | 中性信息、联赛或辅助说明 |

### 3.4 使用规则

- 页面主视觉只保留 1 个产品强调色，避免再次回到紫色科技感。
- 球队色继续只出现在徽章、边框、节点和摘要里，不铺满大面积面板。
- 疲劳、赛程、伤病等模块统一用绿/黄/红三档，不为每个模块重新发明一套颜色。

## 4. 排版与间距

### 4.1 字体建议

- 首选：`"SF Pro Display", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- 保留原版原型的字体风格和层级，不额外做轻量化改字
- 页面主标题和关键数字可以保留 `700-900` 的强对比字重，模块标题和正文维持 `500-700`

### 4.2 字号层级

| 层级 | 建议字号 | 用途 |
|------|----------|------|
| Display | 26-30px | 页面标题、比赛焦点 |
| H1 | 20-22px | 球队名、主要页标题 |
| H2 | 15-17px | 区块标题 |
| Body | 13-15px | 正文、列表内容 |
| Meta | 10-12px | 时间、联赛、次级标签 |

### 4.3 间距规则

- 页面外边距：24px 桌面，16px 移动
- 卡片内边距：20px 桌面，16px 移动
- 卡片圆角：16px
- 小标签圆角：999px
- 分割方式优先使用 1px 边框，不依赖重阴影

## 5. 页面设计

### 5.1 Match Feed

目标：像成熟体育 App 首页一样快速扫读，但信息焦点换成“状态”而不是比分。

结构：

1. 顶部导航
2. 日期切换条：今天 / 明天 / 本周
3. 联赛筛选 chip
4. 比赛列表卡片

每张比赛卡片只展示：

- 联赛名 + 开球时间
- 主客队
- 疲劳等级对比
- 阵容可用率
- 一句极短结论

卡片语气：

- 不写长分析
- 不做大面积插画
- 用户 2 秒内要知道“哪场值得点开”

### 5.2 Match Detail

这是核心页面，按以下顺序排：

1. 比赛头部
2. 双队状态概览
3. 状态总览卡
4. 阵容情况
5. 赛程与走势
6. 关键球员负荷
7. AI 赛前摘要

设计要求：

- 首屏直接完成“谁更累、谁更完整、谁赛程更难”的判断
- 所有复杂信息都做成卡片，不做长滚动图表墙
- 详情页不需要像数据平台一样同时展示十几种图

### 5.3 Player Page

目标：更像“球员工作量简报页”，不是统计百科全书。

结构：

1. 球员概览卡
2. 近 30 天出场负荷
3. 疲劳趋势
4. 伤病记录
5. 同位置对比

视觉上保持和 Match Detail 一致：

- 深色卡片
- 小型图表
- 少量状态标签

### 5.4 World Cup Hub

目标：把 SonarFC 的独特价值放大，但仍然保持页面简洁。

结构：

1. 国家队列表
2. 球队恢复概览
3. 旅行与休整模块
4. 小组赛恢复分析

设计要求：

- 先展示结论，再展示路径
- 地图不是首页主角，更多是辅助模块

## 6. 组件设计

### 6.1 顶部导航

- 保持原版深色顶部栏
- 左侧品牌，右侧为简短产品说明
- 不需要复杂 banner

### 6.2 标签与筛选器

- 默认深色中性底
- 选中为绿色轻渐变底 + 绿色边框
- 高度控制在 32-36px

### 6.3 状态卡

- 深色卡片
- 顶部显示模块标题
- 右侧可放一个轻量标签，例如“High fatigue”
- 内部结构优先行列表，而不是大型图表

### 6.4 疲劳可视化

- 使用水平条或短条形块
- 避免发光、霓虹、重动画
- 文案直接：`高疲劳 / 中负荷 / 状态轻松`

### 6.5 势头模块

- 使用最近 5 场的结果条或方块
- 胜平负颜色清晰，但背景保持克制
- 每个点位只保留必要标签

### 6.6 AI 摘要

- 保持原版次级入口，不抢首屏主结构
- 点击后以弹窗展示两段式摘要：主队一段，客队一段
- 弹窗背景沿用深色面板，只在摘要段落里少量使用球队色

---

## 7. 页面情绪和交互

### 7.1 要保留的感觉

- 像体育 App
- 像可以每天打开的工具
- 像真实产品而不是概念图

### 7.2 要避免的感觉

- 紫色科技感
- 过量霓虹和发光
- 过多动效
- 每个模块都像独立海报

### 7.3 动效原则

- 保留 tab 切换、hover、进度条过渡这些必要动效
- 数字动画可以保留，但只用于疲劳指数等少量关键数值
- 避免重阴影扩散和大面积持续动画

---

## 8. 当前原型改造方向

当前 `docs/sonarfc-prototype.jsx` 的改造原则：

- 保留原版布局、模块顺序和字重层级
- 保留深色信息板的整体气质
- 收掉偏紫的品牌色和高亮方式
- 统一成中性深色底 + 绿色主强调
- 让“颜色更干净”，而不是“结构更简单”
