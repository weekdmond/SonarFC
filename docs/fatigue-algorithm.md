# SonarFC 疲劳指数算法 v3

> 自底向上模型：球员 PFI → 聚合 → 球队 TFI

## 核心理念

球队疲劳不是独立于球员存在的抽象数字，而是由球员的真实身体状态聚合而来。一支球队累不累，取决于它的球员累不累。

```
球员出场分钟 + 恢复曲线 + 年龄 + 位置 → 球员 PFI
                     ↓ 按角色加权平均
球员 PFI × 70% + 赛程压力 × 30% → 球队 TFI
```

输出范围：0-100（0=完全恢复，100=极度疲劳）。前端显示 Energy = 100 - Fatigue。

---

## 1. 球员疲劳指数 (PFI)

### 1.1 角色分类

基于 28 天内的出场模式，每个球员被分为三类：

| 角色 | 判定条件 | 说明 |
|------|---------|------|
| **主力 (starter)** | 28天内首发 ≥ 3次，或首发率 > 60% | 球队核心，疲劳数据最充分 |
| **轮换 (rotation)** | 有出场但不满足主力条件 | 出场不规律，ACWR spike 风险高 |
| **冷板凳 (bench)** | 28天内 0 出场 | 疲劳低，但缺乏比赛节奏 |

### 1.2 五大因子

#### 因子 1: 残余疲劳累积 (Residual Fatigue)

叠加过去 28 天每场比赛的残余疲劳。基于指数衰减恢复曲线：

```
residual = (minutes / 90) × starter_mult × e^(-hours_since / tau)
```

参数：
- `tau` = 20 小时 / age_recovery_factor（年龄越大 tau 越大，恢复越慢）
- `starter_mult` = 1.15（首发）或 1.0（替补）。首发有完整热身 + 赛前准备 + 全场高强度跑动
- 加时赛额外 ×1.3 强度（Carling et al. 2015: CK 峰值高 25-40%）
- 多场比赛的残余疲劳是叠加的，不是只算最后一场

科学依据：
- 90 分钟比赛完全恢复需 72 小时（Nédélec et al. 2012, Sports Medicine）
- 加时赛完全恢复需 96 小时
- 恢复曲线是非线性的，前 24 小时恢复最快

#### 因子 2: 个人 ACWR (Acute:Chronic Workload Ratio)

```
EWMA_acute  = load × 0.25 + (1 - 0.25) × EWMA_prev    (7天窗口)
EWMA_chronic = load × 0.069 + (1 - 0.069) × EWMA_prev  (28天窗口)
ACWR = EWMA_acute / EWMA_chronic
```

风险区间（Gabbett 2016, BJSM）：

| ACWR | 风险 | 说明 |
|------|------|------|
| 0.80 - 1.30 | 低 | Sweet spot，理想负荷区间 |
| < 0.80 | 中 | 训练不足，慢性负荷退化 |
| 1.30 - 1.50 | 中高 | 急性负荷跳升 |
| > 1.50 | 高 | 受伤风险增加 2-4 倍 |

每日负荷计算：比赛日 = 出场分钟 × 位置系数 × 首发系数，非比赛日 = 0。

科学依据：
- Williams et al. (2017, BJSM): EWMA 比 rolling average 对近期负荷变化更敏感
- Blanch & Gabbett (2016): ACWR sweet spot = 0.8-1.3

#### 因子 3: 年龄恢复因子

```python
def age_recovery_factor(age):
    if age <= 22: return 1.05   # 年轻球员恢复略快
    if age <= 27: return 1.0    # 基线
    if age <= 30: return 0.88   # 减慢 12%
    if age <= 33: return 0.78   # 减慢 22%
    if age <= 36: return 0.65   # 减慢 35%
    return 0.50                 # 减慢 50%
```

转换为疲劳分数：`age_score = (1.0 - recovery_factor) × 200`

科学依据：
- 30 岁后肌肉卫星细胞活性下降，VO2max 每年降 0.5-1%
- Carling et al. (2012): 30+ 岁球员密集赛程中 HSR 下降幅度比年轻球员多 8-15%

#### 因子 4: 位置负荷

基于英超分位置跑动数据（Bradley et al. 2009, Bush et al. 2015）：

| 位置 | 系数 | 依据 |
|------|------|------|
| GK 门将 | 0.35 | ~6km/场，最低负荷 |
| DF 后卫 | 0.72 | ~10km，中低负荷 |
| FW 前锋 | 0.82 | ~10.5km，高 HSR 间歇性 |
| MF 中场 | 0.88 | ~12km，最高总距离 |

细分位置（数据支持时）：边锋(1.00) > 边后卫(0.92) > 攻击型中场(0.90) > 中场(0.88) > 影锋(0.82) > 中锋(0.80) > 中后卫(0.68) > 门将(0.35)

#### 因子 5: 赛程压力（球队层面）

由外部传入的球队级别赛程压力分数（见下方 Section 2）。

### 1.3 权重分配（按角色）

| 因子 | 主力 | 轮换 | 冷板凳 |
|------|------|------|--------|
| 残余疲劳 | **40%** | 30% | 10% |
| ACWR | 20% | **30%** | 10% |
| 年龄 | 15% | 15% | **20%** |
| 位置 | 10% | 10% | **15%** |
| 赛程压力 | 15% | 15% | 10% |

设计理由：
- **主力**：出场数据充分，残余疲劳权重最高
- **轮换**：出场不规律，ACWR 更重要（突然上场 = 负荷跳升风险）
- **冷板凳**：无出场数据，用年龄和位置给基线

## 2. 赛程压力因子 (Schedule Pressure)

球队层面的赛程压力，不涉及球员个人数据：

| 因子 | 权重 | 说明 |
|------|------|------|
| 赛程密集度 | 40% | 14 天内比赛数 / 6（6 场 = 100%），叠加最短间隔惩罚 |
| 恢复时间 | 25% | 距上场比赛的指数衰减残余 |
| 旅行惩罚 | 20% | 欧冠客场 +35，欧冠主场 +15，国际赛事 +25 |
| 加时赛 | 15% | 每次加时 +45 |

### 赛程密集度风险系数

基于 Bengtsson et al. (2013, BJSM) 的 UEFA 数据：

| 赛间间隔 | 风险系数 | 受伤风险 |
|---------|---------|---------|
| ≥ 6 天 | 1.0x | 基线 |
| 5 天 | 1.1x | 轻微增加 |
| 4 天 | 1.4x | HSR 下降 3-5% |
| 3 天 | 2.0x | HSR 下降 5-10%，受伤风险翻倍 |
| 2 天 | 3.5x | HSR 下降 10-15% |
| < 2 天 | 5.0-6.0x | 极端情况 |

## 3. 球队疲劳指数 (TFI) — 自底向上聚合

```
TFI = 球员 PFI 加权平均 × 70% + 赛程压力 × 30%
```

### 聚合规则

- **主力**：权重 = 出场分钟 × 3.0（核心阵容决定球队状态）
- **轮换**：权重 = 出场分钟 × 1.5
- **冷板凳**：不参与聚合（没踢球，不影响球队疲劳）

### 为什么 30% 赛程压力？

- 不是所有球员都有完整出场数据，赛程压力弥补信息缺失
- 赛程压力对整队有影响（备战、旅行、训练安排），不只是上场球员

## 4. 数据来源

| 数据 | 来源 | 用途 |
|------|------|------|
| 出场分钟 | API-Football `fixtures/players` | 残余疲劳、ACWR |
| 是否首发 | 每场比赛按分钟数排名 Top 11 | 角色分类、首发强度修正 |
| 球员年龄 | API-Football `players/squads` | 年龄恢复因子 |
| 球员位置 | API-Football (GK/DF/MF/FW) | 位置负荷系数 |
| 比赛日期 | API-Football `fixtures` | 赛程密集度、恢复时间 |
| 主客场 | matches 表 home/away_team_id | 旅行惩罚 |
| 加时赛 | matches 表 has_extra_time | 加时惩罚 |
| 联赛类型 | leagues 表 api_football_id | 欧战/国际赛事判断 |

### 数据限制

我们没有 GPS 数据（总距离、HSR、sprint）、sRPE（主观感知疲劳）、心率数据。所有计算基于公开的比赛出场数据。这意味着：

- 训练负荷无法计入（只能看比赛）
- 同样踢 90 分钟的两个中场，实际跑动量可能差很大
- 伤病恢复期的体能状态无法准确建模

## 5. 未接入数据与演进路线

### 第一层：已有数据，尚未接入算法

| 数据 | 现状 | 对算法的价值 | 优先级 |
|------|------|------------|--------|
| **伤病记录** | `injuries` 表已有 2,883 条 | 伤愈复出球员恢复速度应更慢，长期伤病影响体能基线 | 高 |
| **球员国籍** | `players.nationality` 已有 | 国脚在国际比赛窗口期间有额外出场 + 旅行疲劳 | 中 |

### 第二层：公开数据源可补充

| 数据 | 获取方式 | 对算法的价值 | 优先级 |
|------|---------|------------|--------|
| **球队/城市坐标** | Geocoding API 补 `teams.latitude/longitude` | 计算真实旅行距离（马德里飞莫斯科 vs 马德里去巴塞罗那） | 高 |
| **比赛跑动数据** | FotMob / WhoScored 公开数据 | 替代固定位置系数，用真实场均跑动距离 | 中 |
| **精确赛间间隔** | 已有数据可推算 | 替代 14 天窗口粗估，算每两场之间的精确天数 | 中 |
| **天气数据** | 天气 API | 高温/高湿比赛消耗更大 | 低 |

### 第三层：付费/不公开数据（GPS 级别）

| 数据 | 来源 | 对算法的价值 |
|------|------|------------|
| **高速跑动距离 (HSR)** | STATSports, Catapult | >19.8 km/h 跑动，疲劳预测力最强 |
| **冲刺次数 & 减速次数** | GPS 追踪系统 | 减速 = 肌肉损伤最强标志物（离心收缩） |
| **sRPE (主观感知疲劳)** | 球队内部系统 | 球员自评 RPE × 时长，整合生理+心理疲劳 |
| **心率 & HRV** | 可穿戴设备 | 自主神经系统恢复状态 |
| **睡眠质量** | 可穿戴设备 | 恢复速度的关键因子 |
| **PlayerLoad (Catapult)** | Catapult Vector S7 | 三轴加速度矢量幅度，综合负荷指标 |

### 演进建议

1. **短期（v3.1）** — 接入伤病数据 + 精确赛间间隔。已有数据，改动小，效果明显
2. **中期（v4）** — 补充城市坐标 + FotMob 跑动数据。需要额外数据源但都是公开的
3. **长期（v5）** — 如果与俱乐部合作，接入 GPS/sRPE 数据。模型准确度会有质的飞跃

## 6. 实现

代码位置：`scripts/calculate_fatigue.py`

运行方式：
```bash
python3 scripts/calculate_fatigue.py
```

输出写入 Supabase `fatigue_scores` 表：
- `entity_type`: 'team' 或 'player'
- `entity_id`: teams.id 或 players.id
- `match_id`: 关联的比赛
- `score`: 疲劳指数 (0-100)
- `level`: 'high' (>65) / 'medium' (>40) / 'low'
- `factors`: JSONB，包含各因子明细

前端显示：
- Energy = 100 - score
- 绿色长条 = 高能量（好），红色短条 = 低能量（累）
